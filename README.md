# Jaime de Lucas

Senior Software Engineer focused on backend architecture, Java and AI-assisted software development.

I work on **Context Engineering** — providing AI coding agents with structured, task-scoped project knowledge instead of raw source code, allowing them to spend less effort understanding a codebase and more effort solving engineering problems.

I am building **Romeo**, a platform that turns a codebase into a queryable knowledge model and serves task-scoped context to AI agents. Romeo is proprietary and remains private; I publish only small, self-contained engineering components that demonstrate my engineering practices without exposing the platform itself.

---

## Currently building

**Romeo** is an engineering knowledge platform for AI coding agents.

Instead of asking an agent to read an entire codebase, Romeo builds a structured knowledge model and delivers only the context required for the current task.

The platform itself is proprietary.

The public repositories published here are independent engineering components that demonstrate my engineering practices while protecting Romeo's intellectual property.

---

## Current focus

Building Romeo means working on:

- Context Engineering for AI coding agents
- Knowledge-driven development — treating a codebase as a queryable engineering model rather than a collection of files
- Software architecture and backend engineering
- Measuring the impact of curated context on AI agent productivity

## Public repositories

### Context Cost

A lightweight, dependency-free Java library that estimates the cost reduction achieved by providing an AI coding agent with a carefully curated subset of information instead of an entire corpus.

It represents the public, reusable measurement component behind Romeo. It **measures** the impact of context curation, but it does **not** perform repository scanning, ranking or context selection.

The project demonstrates:

- Clean API design
- Immutable value objects
- Dependency-free JSON parsing
- Command-line interface (CLI)
- Comprehensive JUnit test suite
- Documentation and Architecture Decision Records (ADRs)

→ https://github.com/jaimedelucasdelucas/context-cost

> Romeo's ranking engine, repository scanner, knowledge model, prompts and benchmark internals are intentionally **not** public.
>
> The repositories published here are independent engineering components that demonstrate my engineering practices while protecting proprietary technology.

## Engineering principles

- **Measurable engineering** — improvements should be supported by evidence, not assumptions.
- **Reproducibility** — builds, documentation and results are generated from source whenever possible.
- **Architecture before implementation** — define clear boundaries before writing code.
- **Testing** — unit tests, edge cases and negative scenarios are first-class engineering artifacts.
- **Documentation** — architectural decisions are recorded, interfaces explained and limitations documented transparently.
- **Clean code & maintainability** — small responsibilities, explicit naming and long-term readability over short-term cleverness.
